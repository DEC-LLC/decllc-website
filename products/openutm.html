Hardware Requirements

OpenUTM runs on commodity x86-64 hardware. Requirements scale with your deployment tier.

Component Community Standard Enterprise
CPU 2 cores (x86-64) 4 cores 4+ cores
Memory 3 GB 8 GB 16 GB+
Disk 20 GB 40 GB 80 GB+ (SSD recommended)
Network 2 NICs (uplink + access) 2+ NICs 2+ NICs (10GbE supported)
GPU (AI Plugin) Not required Not required NVIDIA 16 GB+ VRAM
Tesla T4, RTX A4000, Ada 4000 or equivalent
OS Rocky Linux 10 (included in appliance image). Rocky 8 supported for legacy hardware (community tier only, no AI).

AI Plugin — GPU Guide

The AI analysis plugin uses local LLM inference (Ollama + phi4) for firewall rule analysis, security auditing, threat detection, and configuration review. No cloud API calls — all processing stays on your appliance.

Tesla P4 (8 GB)Minimum — quantized models, basic analysis
Tesla T4 (16 GB)Recommended — full phi4 model, real-time analysis
Ada 4000 (20 GB)Ideal — full model + extended context window
A2000 (12 GB)Good mid-range — phi4 at 4-bit quantization

Community and Standard tiers do not require a GPU. The AI plugin is available as an add-on for any tier.
