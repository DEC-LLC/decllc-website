Key Features

Unified Dashboard

See all hosts, workloads, and metrics across every platform in a single web interface. No more switching between vCenter, Proxmox UI, and kubectl.

Agentless by Default

IVMIA collects inventory, metrics, and manages workloads without installing anything on your hypervisors. The core connects directly via SSH and native platform APIs. No agents to deploy, patch, or secure across your fleet.

Dual-Transport Architecture

Every platform supports two transport methods (SSH + API) with automatic failover. If one path goes down, collection continues via the other. Fully agentless on both paths.

Optional Agents for Realtime

Need sub-second metrics or host-level OS data? Deploy lightweight agents (under 100KB, no Java) on specific hosts for higher-frequency collection. Agents are additive — everything works without them.

Federated Multi-Site

Deploy sub-cores at remote sites that collect locally and sync to a central core. Works across VPNs and high-latency links. Each sub-core is also agentless.

Operations Management

Start, stop, restart, snapshot, and manage workloads directly from the web UI. Confirmation dialogs on destructive operations. Full audit trail.

Analysis & Trending

CPU, memory, disk, and network trends over time. Capacity heatmaps. Cross-platform comparison radar charts. All powered by Chart.js with AJAX auto-refresh.

Architecture

Agentless-first design. Agents are optional. Works in heterogeneous and distributed environments.

Agentless Mode (default)

IVMIA Core
  +-- SSH / API to hypervisors
  |     +-- ESXi (vim-cmd + pyVmomi)
  |     +-- Proxmox (pvesh + REST API)
  |     +-- KVM (virsh + TLS)
  |     +-- Hyper-V (PowerShell + WinRM)
  |     +-- Docker (CLI + SDK)
  |     +-- Kubernetes (K8s API + kubectl)
  |     +-- Incus (REST + CLI)
  |     +-- VirtualBox (VBoxManage)
  |
  +-- Web UI + REST API
  +-- PostgreSQL

Nothing to install on target hosts. Inventory, metrics, and operations all work over the network.

With Agents (optional)

IVMIA Core
  +-- Agentless collection (as before)
  +-- Agent check-in API
  |
  +-- Host Agents push metrics
        +-- Higher frequency (every 60s)
        +-- OS-level host metrics
        +-- Auto-register on first run
        +-- RPM / DEB / MSI packages
  |
  +-- Federation (Pro+)
        +-- Sub-Core (remote site)
        +-- Local DB + agentless collectors
        +-- Syncs to central core

Agents add realtime host metrics and higher collection frequency. Everything still works if an agent goes offline.
